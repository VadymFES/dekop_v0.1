## Troubleshooting

### CSP Errors in Browser

If you see CSP violations:
```
Refused to connect to 'https://*.ingest.sentry.io'
```

**Solution:** Already fixed in middleware.ts and next.config.mjs. Restart dev server.

---

### No Data in Sentry Dashboard

Check:
1. `.env.local` has correct SENTRY_DSN
2. `NODE_ENV=production` (dev mode suppresses sends by default)
3. Browser network tab shows requests to `*.ingest.sentry.io`
4. Sentry project ID matches your dashboard

---

### Development Mode Logging

In development, Sentry logs to console but doesn't send to remote:
```typescript
if (process.env.NODE_ENV === "development") {
  console.log("[Sentry Dev]", event.message || event.exception);
  return null; // Don't send
}
```

**To test in development:** Temporarily comment out this check in `sentry.server.config.ts`.
